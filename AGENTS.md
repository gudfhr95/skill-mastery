# Agent Instructions

Before substantial work in this repository, read:

1. `README.md`
2. `docs/INDEX.md`

Keep documentation simple until the project needs more structure. Do not add new
documentation files or folders unless the current work genuinely needs them, and
link any new documentation from `docs/INDEX.md`.

Do not store raw transcripts. Capture only privacy-safe summaries, durable
decisions, and reviewed project context.

Prefer proposal- or pull-request-based skill improvement over direct
self-mutation of trusted skill instructions.

`README.md` is the public project overview. Internal product context belongs
under `docs/`.
